5th prototype iteration

We made changes to the prototype in this version that allow more complicated calculations to work with our service. We created 3 versions or the prototype, 1 for each scenario listed here (access required).

This prototype was tested with users on the week beginning 11 October 2021.

Check NOMIS info page

  • Changes made to sentence information to accommodate more complex calculations:
    • sentences reformatted into a table to handle longer lists of offences more efficiently and allow users to scan more easily
    • text size decreased to accommodate more information on page
    • added verdict column as these are needed for some calculations
    • count number listed for each offence. Consecutive sentences refer to the count they are consecutive to
    • adjustments divided into deductions and additions to make it easier for users to understand days added and subtracted
    • sentences broken down into trial record sheets for calcs where more than one is involved
  • Content changed under the “Some information is wrong” details link, informing users to reload the page when they have changed information in NOMIS. Previous versions suggested the page automatically updated, which is no longer the case.
  • “Last updated from NOMIS” time and date stamp removed as the team felt this could be misleading to users
  • Added a banner when the page has been reloaded to inform users that information may have changed, and to check this before continuing with the calculation

Review calculation page

  • Headings on page changed to Final release dates to be more consistent with PSI
  • Additional dates shown in the Final release dates section for calculations where these are given
  • Hint text added to show users when release dates have been adjusted for weekends
  • Calculation breakdown shown for each sentence where there are calculations with multiple trial record sheets
  • Effective release dates shown for each calculation breakdown where they are calculated separately
  • Content added in inset text to inform users where calculations may have overlapping concurrent sentences that affect the final release dates
  • We have added a journey that allows users to report a problem with the calculation to the Calculate release dates team, which will help improve the calculation engine and allow users to escalate when calculations give unexpected results
  • An option to cancel the calculation, without saving information to NOMIS has been added. This journey also allows users to report a problem as above. This is not mandatory.
