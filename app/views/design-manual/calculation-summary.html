Calculation summary

In design

Help users understand the dates calculated by the service before saving them to NOMIS.

What this page does

This page shows users the final release dates that have been calculated by the service.

Users are shown the release dates and a breakdown of how the service reached these dates. Trust is a key user need of this service, and it is important to gain userâ€™s confidence in the calculation. As a result, the service has been designed to be transparent. The Calculation breakdown clearly shows each stage of the calculation. This has been designed to reflect a manual calculation and to have familiarity, similar to an itemised phone bill or tax document.

Final release dates

In this section, users see the final release dates calculated by the service. These dates include adjustments.

Dates are shown using the release dates summary pattern.

Calculation breakdown

The calculation breakdown shows users how the service reached the final release dates for the calculation. Each stage of the calculation is broken down, showing how release dates or eligibility dates were calculated for each sentence. This mirrors how calculations would be done using a manual calculation sheet.
