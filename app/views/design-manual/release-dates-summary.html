Release dates summary

Pattern Needs more research

Show a persons release dates.

What this pattern does

The release dates summary pattern can be used when showing a prisoner's final calculated release dates.

Dates are displayed in a panel that emphasises them on the page, creating a hierarchy of information.

Any type of final calculated date can be shown. The ordering of dates are determined by the Policy Framework. Labels should be shown with the full name of the date and its ackronym in brackets. For example, "Conditional release date (CRD)".

Hint text

Hint text can be used alongside release dates to show additional relevent information.

For example, when a release date falls on a weekend, the release date is shown as hint text when adjusted to relevent working day in the Calculate release dates service. This has received postive feedback from users.

Hint text should be used sparingly. Overuse could be overwhelming and distract from other content.

Future considerations

The Release dates summary pattern was designed with the intention that it could be used by other services within Digital Prison Services (DPS) and the MoJ where a prisoners releae dates calculated by Calculate release dates would be shown.

Currently there are no APIs that do this, and no work has been done with other services to test how well the design integrates and works.

User testing of this pattern with other services should be carried out and might lead to us changing the design in future.
