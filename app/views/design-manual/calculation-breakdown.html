Calculation breakdown

Pattern Needs more research

Help users understand how release dates were calculated.

What this pattern does

The Calculation breakdown pattern illustrates each stage of a calculation, showing how release dates were worked out by the service.

The pattern can be reused in services that need to show sentence calculations, such as the Manage a digital warrant folder service.

Concurrent sentences

Concurrent sentences are listed in rows with associated information, such as sentence type and date. This helps users understand the calculation rules applied.

For sentence types such as SOPC, the custodial term and licence period are shown for each sentence.

The relevant calculated dates are shown adjacent to each sentence, with the number of days included in the term. This reflects how OMU staff use paper calculation sheets, by counting the number of days in a sentence from the sentence date.

Concurrent sentences shown in the calculation breakdown

Consecutive sentences

Consecutive sentences are shown as an aggregate on the calculation breakdown. They are treated as such in a calculation and when users use paper calculation sheets.

Consecutive sentences are shown as an aggregate in the calculation breakdown

Users can see more information about the sentences included in the aggregate by selecting the “Consecutive sentence details” link. This opens a new page, which shows the details for individual sentences, including notional dates where appropriate.

Individual consecutive sentences are shown on a separate page

Latest release or eligibility dates

In a calculation, the latest relevant release or eligibility dates are used to determine the final release dates, once adjustments are applied. The latest dates relevant to the calculation are highlighted in the calculation breakdown.

Latest release dates highlighted using blue border and label

Release dates that include adjustments

Adjustments that are applied to release or eligibility dates are formatted as a mathematical sum. They show the latest date from the calculation, how an adjustment has been applied, and the new calculated date.

User research

Users found the calculation breakdown page very useful. By seeing a calculation broken down sentence-by-sentence, users reflected how the calculation breakdown mirrors the process of a paper calculation.

Feedback from users included:

"It's good how it doesn’t just produce the dominant date, it splits everything up and breaks it down."

"[It] follows the same order that we do things on the calc sheet. For me this was quite clear. I don't think any case admins, even new ones, could misunderstand this"

Showing how adjustments are applied formatted as a mathematical sum has tested well with users. By using symbols to show addition and subtraction, users have been able to easily understand how the adjustments have been applied, in a way that mirrors their own language.
